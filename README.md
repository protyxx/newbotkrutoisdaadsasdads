# Telegram Bot - Система проверки подписок

Telegram бот для автоматической обработки заявок в канал с проверкой обязательных подписок.

## Возможности

- ✅ Автоматическая обработка заявок на вступление в канал
- 🔍 Проверка подписок на обязательные каналы
- 📊 Статистика для администратора
- 📢 Массовая рассылка сообщений
- 🚀 Оптимизирован для высоких нагрузок (тысячи пользователей)
- 🔒 Защита от спама и взлома (rate limiting)
- 💾 Эффективное хранение данных (SQLite с оптимизацией)

## Установка

1. Установите Python 3.8 или новее

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Настройте бота в файле `config.py`:
   - Получите токен бота у @BotFather
   - Узнайте свой ID у @userinfobot
   - Укажите ID целевого канала
   - Добавьте обязательные каналы для подписки

## Настройка

### 1. Создание бота

1. Напишите @BotFather в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям и получите токен
4. Вставьте токен в `config.py` в переменную `BOT_TOKEN`

### 2. Настройка канала

1. Добавьте бота в ваш канал как администратора
2. Дайте боту права:
   - Добавление участников
   - Просмотр списка участников
3. Получите ID канала (начинается с -100)
4. Укажите ID в `config.py` в переменной `TARGET_CHANNEL_ID`

### 3. Настройка обязательных каналов

В `config.py` отредактируйте список `REQUIRED_CHANNELS`:

```python
REQUIRED_CHANNELS = [
    {
        "id": "@your_channel",  # Для публичных каналов
        "name": "Название канала"
    },
    {
        "id": -1001234567890,  # Для приватных каналов
        "name": "Приватный канал"
    }
]
```

Бот должен быть администратором во всех обязательных каналах!

### 4. ID администратора

Узнайте свой Telegram ID у @userinfobot и укажите в `config.py`:

```python
ADMIN_ID = 123456789
```

## Запуск

## Использование

### Для пользователей

1. Пользователь подает заявку на вступление в канал
2. Бот отправляет ему сообщение со списком обязательных каналов
3. После подписки пользователь нажимает "Проверить подписки"
4. Бот проверяет подписки и автоматически одобряет заявку

### Для администратора

Доступные команды:

- `/stats` - Просмотр статистики бота
- `/broadcast` - Начать рассылку сообщения всем пользователям
- `/cancel` - Отменить текущую операцию

#### Рассылка

1. Отправьте команду `/broadcast`
2. Отправьте сообщение (текст или фото с подписью)
3. Бот разошлет его всем пользователям

## Оптимизация и безопасность

### Производительность

- Многопоточная обработка запросов
- Оптимизированная база данных (WAL режим, индексы)
- Пул соединений для высокой нагрузки
- Асинхронная обработка запросов

### Безопасность

- Rate limiting (защита от спама)
- Cooldown между проверками подписок
- Защита от одновременной обработки запросов
- Валидация прав администратора
- Обработка всех исключений

### Настройки производительности

В `config.py` можно настроить:

```python
RATE_LIMIT_REQUESTS = 10  # Макс. запросов в период
RATE_LIMIT_PERIOD = 60    # Период в секундах
CHECK_SUBSCRIPTION_COOLDOWN = 3  # Задержка между проверками
WORKERS_COUNT = 4  # Количество воркеров
DB_POOL_SIZE = 20  # Размер пула БД
```

## Устранение неполадок

### Бот не отвечает на заявки

- Проверьте, что бот добавлен в канал как администратор
- Убедитесь, что у бота есть права на добавление участников
- Проверьте правильность `TARGET_CHANNEL_ID`

### Не работает проверка подписок

- Бот должен быть администратором во всех обязательных каналах
- Для приватных каналов используйте числовой ID (начинается с -100)
- Для публичных каналов используйте @username

### Ошибки при рассылке

- Некоторые пользователи могли заблокировать бота
- Проверьте лимиты Telegram API (30 сообщений в секунду)
- Бот автоматически добавляет задержки между сообщениями
